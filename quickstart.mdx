## Prerequisites

Before you begin, make sure you have the following installed:

- **Node.js 24+** — the workspace requires Node 24
- **pnpm** — the only supported package manager (npm and yarn are blocked)
- **PostgreSQL** — a running instance with a database you can connect to

<Note>
  This monorepo enforces pnpm via a `preinstall` script. Running `npm install`
  or `yarn` will immediately exit with an error. Install pnpm globally with
  `npm install -g pnpm` before proceeding.
</Note>

## Steps

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/your-org/edulearn-hub.git
    cd edulearn-hub
    ```
  </Step>

  <Step title="Install dependencies">
    Install all workspace packages from the repo root. pnpm resolves the full
    workspace graph in a single pass.

    ```bash
    pnpm install
    ```
  </Step>

  <Step title="Configure the database connection">
    The database layer reads `DATABASE_URL` at startup. Export it in your shell
    or add it to a `.env` file loaded by your environment.

    ```bash
    export DATABASE_URL="postgresql://user:password@localhost:5432/edulearn"
    ```

    <Warning>
      `drizzle.config.ts` will throw an error and refuse to run if
      `DATABASE_URL` is not set.
    </Warning>
  </Step>

  <Step title="Push the database schema">
    Apply the Drizzle schema to your database. In development, use the `push`
    command — no migration files are generated.

    <CodeGroup>
      ```bash Standard
      pnpm --filter @workspace/db run push
      ```

      ```bash Force (reset conflicts)
      pnpm --filter @workspace/db run push-force
      ```
    </CodeGroup>

    Use `push-force` if you have conflicting schema state that needs to be
    overwritten.
  </Step>

  <Step title="Start the API server">
    Run the dev server for `@workspace/api-server`. It uses `tsx` for
    TypeScript-native execution without a separate compile step.

    ```bash
    PORT=3000 pnpm --filter @workspace/api-server run dev
    ```

    You should see:

    ```
    Server listening on port 3000
    ```

    <Note>
      `PORT` is required. The server will throw an error and exit if it is not
      set.
    </Note>
  </Step>

  <Step title="Verify the server is running">
    The health endpoint is mounted at `/api/healthz`. Curl it to confirm the
    server is up and the response matches the `HealthStatus` schema.

    ```bash
    curl http://localhost:3000/api/healthz
    ```

    Expected response:

    ```json
    { "status": "ok" }
    ```
  </Step>
</Steps>

## Next steps

With the server running locally, you can:

- Explore the [project structure](/project-structure) to understand how the monorepo is organized
- Run `pnpm run typecheck` from the root to type-check all packages
- Run `pnpm run build` to produce a production esbuild bundle at `artifacts/api-server/dist/index.cjs`
- Regenerate API client code after editing the OpenAPI spec: `pnpm --filter @workspace/api-spec run codegen`
