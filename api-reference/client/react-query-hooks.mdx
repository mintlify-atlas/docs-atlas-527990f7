The `@workspace/api-client-react` package provides React Query hooks, raw fetch functions, and TypeScript types generated directly from the OpenAPI spec. It is an internal workspace package and is imported via the `workspace:*` protocol in other packages.

## Installation

`@workspace/api-client-react` is a private monorepo package. Add it to another package's `package.json`:

```json
{
  "dependencies": {
    "@workspace/api-client-react": "workspace:*"
  }
}
```

**Peer dependencies:** `react >= 18`, `@tanstack/react-query`

## Types

```ts

export type HealthCheckQueryResult = NonNullable<
  Awaited<ReturnType<typeof healthCheck>>
>;

// Error type thrown on failed requests
export type HealthCheckQueryError = ErrorType<unknown>;
```

`ErrorType<T>` is an alias for `ApiError<T>`, defined in `custom-fetch.ts`. It carries the HTTP status code, status text, response headers, and parsed error body.

## Usage example

```tsx
import { useHealthCheck } from "@workspace/api-client-react";

export function ApiStatus() {
  const { data, isLoading, isError, error } = useHealthCheck();

  if (isLoading) return <p>Checking API health…</p>;

  if (isError) {
    return <p>API unavailable: {error.message}</p>;
  }

  return <p>API status: {data.status}</p>;
}
```

## Customizing fetch behavior

All generated hooks delegate to `customFetch` from `lib/api-client-react/src/custom-fetch.ts`. To add global behavior such as auth headers, base URL rewriting, or request logging, modify `custom-fetch.ts` directly — it is **not** a generated file and is safe to edit.

```ts
// custom-fetch.ts (safe to edit)
export async function customFetch<T = unknown>(
  input: RequestInfo | URL,
  options: CustomFetchOptions = {},
): Promise<T>
```

<Tip>
  To add an `Authorization` header to every request, intercept the call inside `customFetch` and merge your header into the `options` before the `fetch()` call.
</Tip>
