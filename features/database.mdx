The `@workspace/db` package provides a shared PostgreSQL client and schema definitions used across the monorepo. It is built on [Drizzle ORM](https://orm.drizzle.team/) with the `node-postgres` (`pg`) adapter.

## Client setup

The package entry point creates two exports: a raw `Pool` for direct query access and a typed `db` client that knows about your schema:

```ts src/index.ts
import { drizzle } from "drizzle-orm/node-postgres";
import pg from "pg";
import * as schema from "./schema";

const { Pool } = pg;

if (!process.env.DATABASE_URL) {
  throw new Error(
    "DATABASE_URL must be set. Did you forget to provision a database?",
  );
}

export const pool = new Pool({ connectionString: process.env.DATABASE_URL });
export const db = drizzle(pool, { schema });

export * from "./schema";
```

The module throws at import time if `DATABASE_URL` is not set, so misconfigured environments fail loudly rather than at the first query.

## Schema barrel export

All table definitions live under `src/schema/` and are re-exported through the barrel file `src/schema/index.ts`. The convention is one file per model:

```ts src/schema/index.ts
// Export your models here. Add one export per file
// export * from "./posts";
//
// Each model/table should ideally be split into different files.
// Each model/table should define a Drizzle table, insert schema, and types:
//
//   import { pgTable, text, serial } from "drizzle-orm/pg-core";
//   import { createInsertSchema } from "drizzle-zod";
//   import { z } from "zod/v4";
//
//   export const postsTable = pgTable("posts", {
//     id: serial("id").primaryKey(),
//     title: text("title").notNull(),
//   });
//
//   export const insertPostSchema = createInsertSchema(postsTable).omit({ id: true });
//   export type InsertPost = z.infer<typeof insertPostSchema>;
//   export type Post = typeof postsTable.$inferSelect;

export {}
```

Because `src/index.ts` does `export * from "./schema"`, all table types and Zod insert schemas are available from the root import (`@workspace/db`).

## Drizzle config

`drizzle.config.ts` points Drizzle Kit at the schema barrel and the live database:

```ts drizzle.config.ts
import { defineConfig } from "drizzle-kit";
import path from "path";

if (!process.env.DATABASE_URL) {
  throw new Error("DATABASE_URL, ensure the database is provisioned");
}

export default defineConfig({
  schema: path.join(__dirname, "./src/schema/index.ts"),
  dialect: "postgresql",
  dbCredentials: {
    url: process.env.DATABASE_URL,
  },
});
```

`DATABASE_URL` must be set whenever Drizzle Kit commands are run.

## Adding a new table

<Steps>
  <Step title="Create the schema file">
    Add a file at `src/schema/<modelname>.ts`. Define the Drizzle table,
    a Zod insert schema (via `drizzle-zod`), and the inferred TypeScript types:

    ```ts src/schema/courses.ts
    import { pgTable, text, serial, timestamp } from "drizzle-orm/pg-core";
    import { createInsertSchema } from "drizzle-zod";
    import { z } from "zod/v4";

    export const coursesTable = pgTable("courses", {
      id: serial("id").primaryKey(),
      title: text("title").notNull(),
      slug: text("slug").notNull().unique(),
      createdAt: timestamp("created_at").defaultNow().notNull(),
    });

    export const insertCourseSchema = createInsertSchema(coursesTable).omit({
      id: true,
      createdAt: true,
    });
    export type InsertCourse = z.infer<typeof insertCourseSchema>;
    export type Course = typeof coursesTable.$inferSelect;
    ```
  </Step>

  <Step title="Re-export from the barrel">
    Add a line to `src/schema/index.ts`:

    ```ts src/schema/index.ts
    export * from "./courses";
    ```
  </Step>

  <Step title="Push the schema to the database">
    Run the push command to apply your changes to the connected database:

    ```bash
    pnpm --filter @workspace/db run push
    ```
  </Step>
</Steps>

<Warning>
  `drizzle-kit push` introspects your running database and applies schema
  changes directly without generating migration files. It is intended for
  development only. Do not run it against a production database — use
  a proper migration workflow (`drizzle-kit generate` + `drizzle-kit migrate`)
  for production deployments.
</Warning>

## Package exports

The package exposes two export conditions defined in `package.json`:

| Export path | Contents |
|---|---|
| `.` (default) | `pool`, `db`, full schema re-exports |
| `./schema` | Schema tables, insert schemas, and types only |

Use the `./schema` sub-path in packages that only need type information and do not require a live database connection.

## Development commands

```bash
# Push schema changes to the development database
pnpm --filter @workspace/db run push

# Force push (skips confirmation prompts for destructive changes)
pnpm --filter @workspace/db run push-force
```
